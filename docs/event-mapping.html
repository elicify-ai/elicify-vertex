When Fablize vs Vertex inject prompts

Updated for @elicify-ai/elicify-vertex@0.9.3+: static contract lives in the agent prompt and the /elicify-vertex slash template. Per-turn system inject is dynamic only (mode, ledger, queued events) — not a full contract replay.

Where the static “always-on” home lives

Fablize

  • Home: short block written into CLAUDE.md at install
  • When loaded: every Claude session that uses that CLAUDE.md
  • Not re-pasted by a hook every API call
  • Long packs stay as files; router only adds a one-liner pointer

Vertex (0.9.3+)

  • Home (agent): full [vertex:contract] inside elicify-vertex-agent.md
  • Home (slash): same contract inside /elicify-vertex template — slash injects behavior, not only a flag
  • Not re-pasted every turn via system.transform
  • Per-turn system: mode / ledger / queued failures & stops only
